Project Scheduler Demo

Информация: Эта демо-страница показывает работу планировщика проектов. В реальной системе скрипт будет загружать данные через API, но здесь мы используем тестовые данные.
Внимание: Для работы с реальным API необходимо:
  • Разместить скрипт на странице системы integram.io
  • Убедиться, что переменные db и xsrf доступны
  • Настроить правильные ID отчётов в конфигурации

Использование в реальной системе

Для использования скрипта в системе integram.io:

  1. Скопируйте содержимое файла project-scheduler.js
  2. Вставьте его на страницу через тег <script>
  3. Убедитесь, что на странице есть div с классом content
  4. Скрипт автоматически загрузит данные и отобразит график

Пример вставки скрипта:

<div class="content"></div>
<script src="project-scheduler.js"></script>

Возможности планировщика:

  • ✓ Очистка данных проекта перед планированием (опционально)
  • ✓ Автоматический расчёт длительности на основе шаблонных нормативов
  • ✓ Учёт количества при расчёте длительности
  • ✓ Сохранение рассчитанной длительности в систему
  • ✓ Планирование с учётом рабочего времени (9:00-18:00)
  • ✓ Учёт обеденного перерыва (13:00-14:00)
  • ✓ Соблюдение зависимостей между задачами и операциями
  • ✓ Задачи ≤ 4 часов не разбиваются на разные дни
  • ✓ Сохранение времени начала в систему
  • ✓ Отображение графика в виде таблицы

Конфигурация:

Основные параметры в начале файла project-scheduler.js:

const CONFIG = {
    // API endpoints
    projectReportId: 2681,           // ID отчёта с данными проекта
    workHoursReportId: 3283,         // ID отчёта с настройками рабочего времени
    cleanupReportId: 2310,           // ID отчёта для очистки данных проекта

    // Field codes
    taskDurationCode: 't3758',       // Код параметра длительности задачи
    operationDurationCode: 't3757',  // Код параметра длительности операции
    taskStartCode: 't798',           // Код параметра времени начала задачи
    operationStartCode: 't2665',     // Код параметра времени начала операции

    // Default values
    defaultDuration: 60,             // Длительность по умолчанию (минуты)
    lunchDuration: 60,               // Длительность обеда (минуты)

    // Cleanup options
    enableCleanup: true              // Включить очистку данных перед планированием
};

Очистка данных проекта:

Перед запуском планирования система может автоматически очистить данные проекта:

  • Удаляются все даты начала задач и операций
  • Удаляются все рассчитанные длительности
  • Удаляются все назначения исполнителей

Для включения/выключения этой функции используйте параметр enableCleanup в конфигурации:

  • enableCleanup: true - очистка включена (рекомендуется)
  • enableCleanup: false - очистка отключена

Примечание: Очистка использует отчёт с ID 2310, который принимает параметр FR_ProjID (ID проекта).

Тестирование:

Для проверки корректности работы запустите тестовый скрипт:

node test-scheduler.js

Приоритет источников нормативов:

  1. Существующий норматив - если в проекте уже указана длительность > 0
  2. Норматив из шаблона × количество - берётся норматив из шаблонного проекта и умножается на количество
  3. Значение по умолчанию - если норматив не найден в шаблоне (60 минут)

Обработка данных:

  • Количество: Если пустое, 0 или нечисловое → считается = 1
  • Дата старта: Формат "dd.mm.yyyy" (например, "20.11.2025")
  • Время начала: Формат "dd.mm.yyyy HH:MM:SS" (например, "29.11.2025 17:39:00")

Проверка перед сдачей:

Тест 1: Нормативы

  • ✓ Задача без норматива получает норматив из шаблона
  • ✓ Операция без норматива получает норматив из шаблона
  • ✓ Умножение на количество работает корректно
  • ✓ Существующие нормативы не перезаписываются

Тест 2: Время

  • ✓ Дата старта проекта парсится корректно
  • ✓ Время не назначается на нерабочие часы
  • ✓ Обеденный перерыв учитывается
  • ✓ Зависимости между задачами соблюдаются

Тест 3: Отображение

  • ✓ В таблице нет NaN или undefined
  • ✓ Даты отображаются в читаемом формате
  • ✓ Длительность рассчитывается корректно
  • ✓ Статусы нормативов понятны
